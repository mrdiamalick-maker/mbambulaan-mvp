# Strategie IA modulaire et gouvernance - Mbambulaan

## Decision produit

L'IA Mbambulaan est un vrai levier de differenciation, mais elle doit etre modulaire, controlable et rassurante.

Le bon positionnement : l'IA assiste, l'humain valide.

## Principe cle

Chaque module IA doit pouvoir etre active ou desactive selon :

- le profil partenaire ;
- le niveau de sensibilite des donnees ;
- les permissions internes ;
- le niveau de confiance souhaite ;
- le mode de deploiement ;
- le cadre contractuel.

## Pourquoi c'est important

Les institutions peuvent etre prudentes sur la confidentialite, la souverainete, les donnees sensibles et les usages automatises.

Mbambulaan doit donc montrer que l'IA n'est pas une boite noire imposee. C'est une couche optionnelle, parametree, journalisee et validee par l'humain.

## Catalogue IA activable

### IA Synthese

Produit des resumes de situation a partir des donnees autorisees.

Activation : standard avance / institutionnel.

Risque : moyen.

Controle : validation humaine avant diffusion.

### IA Territoire

Resume un territoire, ses tensions, programmes, incidents, budgets et preuves.

Activation : institutionnel.

Risque : moyen.

Controle : donnees sources affichees.

### IA Alertes

Detecte signaux faibles, retards, ecarts budgetaires, incidents ouverts, ressources critiques.

Activation : optionnelle.

Risque : eleve si donnees sensibles.

Controle : seuils parametrables et journal d'alerte.

### IA Notes et rapports

Prepare notes d'arbitrage, points de suivi et rapports.

Activation : institutionnel.

Risque : eleve.

Controle : brouillon uniquement, validation humaine obligatoire.

### IA Doublons programmes

Detecte chevauchements entre programmes, partenaires, territoires ou financements.

Activation : institutionnel.

Risque : moyen.

Controle : suggestion, jamais decision automatique.

### IA Recherche assistee

Permet de poser des questions sur les donnees autorisees.

Activation : optionnelle.

Risque : moyen a eleve selon les donnees.

Controle : acces par role et historique des requetes.

## Parametres IA par partenaire

Un espace partenaire doit pouvoir afficher :

- modules IA actifs ;
- modules IA inactifs ;
- niveau de sensibilite ;
- roles autorises ;
- historique des generations ;
- statut : brouillon, valide, archive ;
- option de desactivation.

## Message commercial recommande

L'IA Mbambulaan aide les equipes a synthetiser, analyser et preparer des decisions a partir des donnees autorisees. Les modules IA sont activables ou desactivables, les acces sont controles par role et les decisions restent humaines.

## Regles de gouvernance

1. Pas de decision automatique sans validation humaine.
2. Pas d'IA sur un module non autorise.
3. Pas de generation sans afficher les donnees sources ou le contexte.
4. Les contenus generes restent des brouillons tant qu'ils ne sont pas valides.
5. Les actions sensibles sont journalisees.
6. Chaque partenaire peut choisir les modules IA actives.
7. Les modules IA doivent pouvoir etre desactives.

## Implication produit

Dans l'espace Ministere, ajouter une zone Parametres IA ou Gouvernance IA dans le module Notes, preuves et acces.

Afficher :

- IA Synthese : active ;
- IA Alertes budgetaires : active ;
- IA Notes : brouillon uniquement ;
- IA Recherche : desactivee ;
- validation humaine : obligatoire ;
- historique : disponible.

## Implication commerciale

L'IA devient un module premium, pas une promesse vague.

Elle peut etre vendue par niveau :

- IA basique : synthese et questions simples ;
- IA institutionnelle : alertes, notes, rapports, priorisation ;
- IA avancee : connecteurs, recherche assistee, modeles dedies, deploiement dedie.

## Risques a surveiller

- impression de surveillance ;
- peur de fuite de donnees ;
- confusion entre suggestion et decision ;
- donnees obsoletes qui produisent une mauvaise synthese ;
- demande de garanties techniques trop tot.

## Reponse Mbambulaan

Modularite, permissions, journalisation, validation humaine, choix cloud ou deploiement dedie a terme.
