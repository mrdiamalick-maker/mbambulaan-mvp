# 09 — KPI et modèle d'impact

## 1. Finalité

Ce document définit comment Mbàmbulaan mesure son utilité, son adoption, sa capacité de coordination, ses résultats et sa création de valeur économique.

Le modèle de mesure doit éviter deux dérives :

- accumuler des indicateurs sans lien avec les décisions ;
- présenter des chiffres d'activité comme une preuve suffisante d'impact.

Mbàmbulaan doit mesurer une chaîne complète :

```text
Usage
→ Coordination
→ Exécution
→ Résultats observables
→ Effets pour les acteurs
→ Valeur économique et institutionnelle
```

## 2. Clarification terminologique : ne pas utiliser « preuve » comme terme métier générique

Le terme « preuve » peut être mal compris par les acteurs de la filière.

Dans les interfaces et les échanges métier, il doit être remplacé par un vocabulaire concret selon le contexte.

### Termes recommandés

- **élément de confirmation** : photo, document ou validation permettant de confirmer une information ;
- **justificatif** : document attendu pour une action formelle ou financière ;
- **constat** : information relevée sur le terrain ;
- **pièce jointe** : photo ou document ajouté à une situation ;
- **confirmation d'exécution** : élément montrant qu'une action a été réalisée ;
- **validation** : confirmation donnée par une personne habilitée ;
- **source** : origine d'une information ;
- **trace** : historique conservé dans le système.

### Règle produit

Dans le modèle technique, l'objet interne peut conserver un concept générique de type `Evidence` ou `Preuve` si nécessaire.

Dans les parcours utilisateurs, les libellés doivent être adaptés au métier.

Exemples :

- « Ajouter une photo ou un document » plutôt que « Ajouter une preuve » ;
- « Comment confirmer que l'action est terminée ? » plutôt que « Quelle preuve est attendue ? » ;
- « Joindre un justificatif » pour un décaissement ;
- « Faire valider l'intervention » pour une action institutionnelle.

## 3. Principes de mesure

Les indicateurs doivent être :

- utiles à une décision ;
- compréhensibles ;
- calculables de manière fiable ;
- limités en nombre ;
- segmentables par territoire, type d'acteur et organisation ;
- comparables dans le temps ;
- accompagnés de leur source et de leur méthode de calcul.

## 4. Niveaux du modèle d'impact

### Niveau 1 — Activité

Mesure ce qui est fait dans le produit.

Exemples :

- utilisateurs actifs ;
- signalements soumis ;
- situations qualifiées ;
- décisions enregistrées ;
- engagements créés ;
- actions assignées.

### Niveau 2 — Coordination

Mesure l'amélioration du fonctionnement collectif.

Exemples :

- délai de prise en charge ;
- délai de qualification ;
- délai de décision ;
- taux d'engagements acceptés ;
- nombre de blocages identifiés ;
- nombre de relances nécessaires.

### Niveau 3 — Exécution

Mesure la réalisation effective des actions.

Exemples :

- taux d'actions terminées ;
- taux d'actions vérifiées ;
- taux d'engagements tenus ;
- taux de situations clôturées ;
- délai moyen de résolution.

### Niveau 4 — Résultats

Mesure ce qui change concrètement.

Exemples :

- équipement remis en service ;
- réduction du temps d'indisponibilité ;
- amélioration de la salubrité ;
- réduction des pertes ;
- intervention publique mieux coordonnée.

### Niveau 5 — Effets

Mesure les bénéfices pour les acteurs.

Exemples :

- confiance accrue ;
- meilleure visibilité ;
- baisse des coûts de coordination ;
- meilleure capacité de décision ;
- réduction des conflits ;
- meilleure continuité d'activité.

### Niveau 6 — Valeur économique et institutionnelle

Mesure la capacité de Mbàmbulaan à créer et capter durablement de la valeur.

Exemples :

- organisation prête à payer ;
- pilote financé ;
- coût par situation traitée ;
- coût par territoire ;
- revenu récurrent potentiel ;
- réduction des coûts pour le payeur ;
- contribution à la gouvernance sectorielle.

## 5. North Star Metric

La North Star Metric recommandée pour le pilote est :

> Nombre de situations réelles coordonnées jusqu'à une issue vérifiée et acceptée par les acteurs concernés.

Cette métrique combine :

- usage ;
- coordination ;
- exécution ;
- résultat ;
- confiance.

Elle évite de valoriser uniquement le volume de signalements.

## 6. KPI d'adoption

### Utilisateurs activés

Définition : utilisateur ayant complété son accès et réalisé une première action utile.

### Utilisateurs actifs hebdomadaires

Définition : utilisateurs ayant réalisé au moins une action métier sur sept jours.

### Taux d'activation

```text
Utilisateurs activés / Utilisateurs invités
```

### Taux de rétention

Mesure la proportion d'utilisateurs qui reviennent après leur première utilisation.

### Couverture des acteurs clés

Mesure la représentation effective des catégories d'acteurs nécessaires au flux pilote.

## 7. KPI de coordination

### Délai de prise en charge

Temps entre la soumission d'un signalement et sa première prise en charge.

### Délai de qualification

Temps entre la soumission et la qualification de la situation.

### Délai de décision

Temps entre la qualification et la décision opérationnelle.

### Taux de responsabilités explicites

```text
Objets opérationnels avec responsable identifié / Objets opérationnels actifs
```

### Taux d'engagements acceptés

```text
Engagements acceptés / Engagements proposés
```

### Taux de blocages avec propriétaire

```text
Blocages avec responsable de résolution identifié / Blocages déclarés
```

## 8. KPI d'exécution

### Taux d'actions terminées à temps

```text
Actions terminées avant échéance / Actions arrivées à échéance
```

### Taux d'actions confirmées

Définition : proportion d'actions terminées disposant d'un élément de confirmation adapté au contexte.

### Taux d'engagements tenus

```text
Engagements tenus / Engagements arrivés à échéance
```

### Taux de situations clôturées

```text
Situations clôturées / Situations ouvertes sur la période
```

### Délai moyen de résolution

Temps entre la création de la situation et sa clôture.

## 9. KPI de confiance

### Taux d'informations sourcées

Mesure la proportion d'informations importantes dont l'origine est connue.

### Taux de décisions justifiées

Mesure la proportion de décisions accompagnées d'une justification compréhensible.

### Taux d'actions confirmées par un tiers

Mesure la proportion d'actions vérifiées par une personne habilitée autre que l'exécutant.

### Taux de clôtures contestées

```text
Clôtures contestées / Situations clôturées
```

### Indice de confiance perçue

Mesuré par enquête courte auprès des acteurs.

## 10. KPI d'utilité métier

### Situations réellement résolues

Mesure le nombre de situations ayant abouti à une amélioration concrète.

### Temps de coordination économisé

Estimation du temps réduit sur :

- appels ;
- déplacements ;
- relances ;
- recherche d'information ;
- comptes rendus manuels.

### Coûts évités

Exemples :

- durée d'indisponibilité réduite ;
- pertes évitées ;
- interventions en doublon évitées ;
- retards administratifs réduits.

### Taux de réutilisation

Mesure la proportion d'organisations qui utilisent Mbàmbulaan sur plusieurs situations.

## 11. KPI économiques

### Coût d'acquisition d'une organisation

Inclut :

- prospection ;
- démonstration ;
- contractualisation ;
- onboarding.

### Coût de déploiement par territoire

Inclut :

- configuration ;
- formation ;
- accompagnement ;
- support ;
- adaptation locale.

### Coût par situation traitée

```text
Coûts opérationnels du pilote / Situations traitées jusqu'à clôture
```

### Revenu potentiel par territoire

Projection du revenu annuel possible selon le modèle retenu.

### Marge brute potentielle

```text
Revenus - coûts directs de service
```

### Volonté de paiement

Mesurée par :

- budget identifié ;
- lettre d'intérêt ;
- financement du pilote ;
- engagement contractuel ;
- renouvellement.

## 12. KPI institutionnels

- nombre d'organisations publiques actives ;
- nombre de décisions institutionnelles suivies ;
- nombre de coordinations inter-organisations ;
- taux de participation des acteurs clés ;
- fréquence d'utilisation par les décideurs ;
- capacité à produire une synthèse territoriale fiable.

## 13. Indicateurs de risque

### Risque d'outil de remontée sans action

```text
Signalements sans décision ni action / Signalements soumis
```

### Risque de dépendance au coordinateur

Mesure la concentration des actions sur une seule personne.

### Risque de faible confiance

Mesure les clôtures contestées, corrections et informations non sourcées.

### Risque de surcharge

Mesure les files d'attente, délais et éléments sans responsable.

### Risque économique

Mesure le coût de support, le temps d'onboarding et l'absence de budget identifié.

## 14. Tableau de bord pilote

Le tableau de bord du pilote doit rester limité à cinq blocs :

### Adoption

- utilisateurs actifs ;
- organisations actives ;
- taux d'activation.

### Coordination

- délai de prise en charge ;
- délai de qualification ;
- engagements acceptés.

### Exécution

- actions à temps ;
- engagements tenus ;
- situations clôturées.

### Confiance

- décisions justifiées ;
- actions confirmées ;
- clôtures contestées.

### Économie

- coût par situation ;
- coût par territoire ;
- budget ou revenu identifié.

## 15. Méthodes de collecte

### Données automatiques

Issues des événements métier et journaux du produit.

### Données déclaratives

Issues de formulaires ou mises à jour des acteurs.

### Données de validation

Issues d'une personne habilitée confirmant une action ou un résultat.

### Enquêtes

Courtes, ciblées et liées à une situation récente.

### Entretiens

Utilisés pour comprendre les mécanismes, pas seulement produire un score.

## 16. Qualité des indicateurs

Chaque KPI doit préciser :

- définition ;
- formule ;
- source ;
- propriétaire ;
- fréquence ;
- segmentation ;
- seuil d'alerte ;
- décision associée ;
- limites d'interprétation.

## 17. Baseline

Avant le pilote, mesurer si possible :

- délais actuels ;
- nombre de relances ;
- outils utilisés ;
- taux de résolution ;
- coûts visibles ;
- niveau de satisfaction ;
- qualité de traçabilité.

Sans baseline, Mbàmbulaan peut mesurer son activité mais difficilement démontrer l'amélioration.

## 18. Seuils pilote proposés

### Adoption

- 30 à 50 utilisateurs actifs ;
- 3 à 5 organisations actives ;
- au moins 60 % d'activation des utilisateurs invités.

### Coordination

- 60 % des signalements pris en charge en moins de 24 heures ;
- 60 % des situations qualifiées en moins de 48 heures ;
- 70 % des objets actifs avec un responsable clair.

### Exécution

- 60 % des actions terminées ou justifiées ;
- 60 % des engagements tenus ou replanifiés explicitement ;
- 50 % des situations clôturées dans le délai cible.

### Confiance

- 80 % des décisions avec justification ;
- 70 % des actions critiques avec confirmation ;
- moins de 10 % de clôtures contestées.

### Économie

- au moins un budget identifié ;
- au moins une organisation prête à financer la suite ;
- coût par situation suffisamment bas pour permettre un modèle soutenable.

## 19. Règles d'interprétation

Un bon volume d'usage ne suffit pas si :

- les situations ne progressent pas ;
- les responsabilités restent floues ;
- les actions ne sont pas exécutées ;
- aucun acteur ne finance la suite.

Inversement, un faible volume peut être pertinent si les situations traitées sont critiques et génèrent une forte valeur.

## 20. Décisions pilotées par les KPI

Les KPI doivent permettre de décider :

- poursuivre le pilote ;
- réduire le périmètre ;
- changer de territoire ;
- renforcer l'accompagnement ;
- revoir les rôles ;
- simplifier le produit ;
- modifier le modèle économique ;
- arrêter une capacité peu utilisée.

## 21. Principe directeur

> Mbàmbulaan ne doit pas seulement compter des usages. Il doit démontrer que la coordination s'améliore, que des situations aboutissent à une issue confirmée et qu'un acteur reconnaît une valeur suffisante pour financer durablement le service.
