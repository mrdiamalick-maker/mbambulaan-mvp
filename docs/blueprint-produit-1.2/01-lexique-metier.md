# 01 — Lexique métier Mbàmbulaan

## 1. Finalité

Ce document définit le vocabulaire officiel de Mbàmbulaan.

Il sert de référence commune pour :

- les équipes produit ;
- les équipes techniques ;
- les designers ;
- les partenaires institutionnels ;
- les acteurs de terrain ;
- les supports de formation ;
- les documents contractuels ;
- les interfaces utilisateurs.

L'objectif est d'éviter les ambiguïtés, les mots trop techniques et les interprétations différentes d'un même concept.

## 2. Principes de langage

Le vocabulaire Mbàmbulaan doit être :

- simple ;
- concret ;
- compréhensible sans formation technique ;
- fidèle aux réalités métier ;
- cohérent dans tout le produit ;
- traduisible en wolof et, à terme, dans d'autres langues nationales ;
- suffisamment précis pour guider le développement.

## 3. Règle de double langage

Mbàmbulaan peut distinguer :

### Langage interne produit et technique

Utilisé dans :

- le modèle de domaine ;
- les APIs ;
- la base de données ;
- les événements ;
- la documentation technique.

### Langage utilisateur

Utilisé dans :

- les écrans ;
- les formulaires ;
- les notifications ;
- les formations ;
- les échanges avec les acteurs.

Le langage utilisateur doit toujours primer dans l'expérience.

## 4. Termes fondamentaux

### Acteur

Personne qui intervient dans la filière ou dans la coordination d'une situation.

Exemples :

- pêcheur ;
- mareyeur ;
- transformatrice ;
- responsable de site ;
- agent public ;
- collectivité ;
- technicien ;
- partenaire financier.

À éviter dans les interfaces :

- utilisateur final ;
- compte ;
- ressource humaine.

### Organisation

Structure à laquelle un acteur est rattaché ou au nom de laquelle il agit.

Exemples :

- GIE ;
- association ;
- service de l'État ;
- collectivité ;
- entreprise ;
- ONG ;
- projet ;
- comité de gestion.

### Rôle

Fonction reconnue à un acteur dans le produit.

Exemples :

- déclarant ;
- coordinateur ;
- décideur ;
- exécutant ;
- validateur ;
- administrateur.

### Mandat

Autorisation donnée à un acteur pour agir dans un périmètre donné.

Un mandat précise :

- ce que l'acteur peut faire ;
- sur quel territoire ;
- au nom de quelle organisation ;
- pendant quelle période.

Dans l'interface, le mot « autorisation » peut être préféré selon le public.

## 5. Termes liés au territoire

### Territoire

Zone géographique ou espace d'intervention auquel sont rattachés les acteurs, situations et actions.

Exemples :

- quai de pêche ;
- site de transformation ;
- commune ;
- département ;
- région ;
- zone maritime.

### Site

Lieu précis où se déroule une activité ou une situation.

Exemples :

- quai ;
- aire de transformation ;
- marché ;
- fabrique de glace ;
- débarcadère.

## 6. Termes liés à la remontée terrain

### Signalement

Information transmise pour attirer l'attention sur un problème, un besoin ou une situation.

Un signalement peut être incomplet au moment de sa création.

Exemples d'interface :

- « Signaler un problème » ;
- « Signaler un besoin » ;
- « Faire une remontée ».

### Observation

Information relevée sur le terrain sans qu'elle soit nécessairement un problème.

Exemples :

- changement de pratique ;
- évolution d'une activité ;
- constat sanitaire ;
- information sur un équipement.

### Constat

Fait observé et décrit de manière concrète.

Le constat peut être appuyé par :

- une photo ;
- un document ;
- une validation ;
- une source identifiée.

### Situation

Ensemble structuré d'informations issu d'un ou plusieurs signalements et nécessitant une coordination, une décision ou une action.

Un signalement devient une situation lorsqu'il a été qualifié.

### Besoin

Écart entre la situation actuelle et ce qui est nécessaire pour fonctionner correctement.

Exemples :

- besoin de réparation ;
- besoin de glace ;
- besoin d'équipement ;
- besoin de formation ;
- besoin d'intervention publique.

### Besoin priorisé

Besoin dont le niveau d'importance a été confirmé selon des critères explicites.

## 7. Termes liés à l'analyse

### Qualification

Action qui consiste à analyser une remontée pour déterminer :

- sa nature ;
- son urgence ;
- sa gravité ;
- les acteurs concernés ;
- l'orientation à donner.

Dans l'interface, on peut utiliser :

- « Analyser la situation » ;
- « Compléter l'analyse » ;
- « Confirmer la priorité ».

### Urgence

Niveau indiquant dans quel délai une intervention est nécessaire.

### Criticité

Niveau indiquant l'importance des conséquences si la situation n'est pas traitée.

### Priorité

Ordre dans lequel une situation doit être traitée, sur la base de critères connus.

Une priorité n'est pas uniquement une urgence.

## 8. Termes liés à la décision

### Décision

Choix formalisé par une personne ou une instance habilitée.

Une décision doit préciser :

- qui décide ;
- ce qui est décidé ;
- pourquoi ;
- les conséquences ;
- les responsables désignés.

### Arbitrage

Choix entre plusieurs options, priorités ou intérêts contradictoires.

### Validation

Confirmation donnée par une personne habilitée.

Exemples :

- validation d'une action ;
- validation d'un document ;
- validation d'une clôture ;
- validation d'un financement.

### Refus

Décision de ne pas accepter une proposition, une demande ou une validation.

Le motif du refus doit être explicite.

## 9. Termes liés à la coordination

### Coordination

Organisation des acteurs, responsabilités, informations et actions nécessaires pour faire progresser une situation.

### Espace de coordination

Espace partagé dans lequel les acteurs suivent une situation, prennent des décisions et organisent les actions.

Dans l'interface, des formulations plus simples peuvent être utilisées :

- « Suivi de la situation » ;
- « Espace de travail » ;
- « Coordination en cours ».

### Participant

Acteur impliqué dans une coordination.

### Responsable

Acteur chargé de faire progresser un élément précis.

### Bénéficiaire

Acteur ou groupe qui tire un bénéfice direct d'une action ou d'une initiative.

### Partie prenante

Acteur concerné par une situation sans être nécessairement responsable d'une action.

À utiliser surtout dans la documentation, moins dans les interfaces terrain.

## 10. Termes liés aux engagements et actions

### Engagement

Promesse explicite faite par un acteur ou une organisation.

Un engagement précise :

- qui s'engage ;
- sur quoi ;
- envers qui ;
- avant quand ;
- comment l'exécution sera confirmée.

### Action

Tâche concrète à réaliser pour faire progresser une situation ou tenir un engagement.

### Intervention

Action menée sur le terrain ou par un service compétent.

Exemples :

- réparation ;
- contrôle ;
- nettoyage ;
- livraison ;
- formation ;
- déplacement d'une équipe.

### Échéance

Date ou délai attendu pour terminer une action ou tenir un engagement.

### Blocage

Élément empêchant une action ou une situation de progresser.

Un blocage doit préciser :

- la cause ;
- l'impact ;
- le responsable de résolution ;
- l'action attendue.

### Retard

Dépassement d'une échéance sans qu'un obstacle formel soit nécessairement déclaré.

### Relance

Rappel envoyé à un acteur afin qu'il réalise ou mette à jour une action attendue.

## 11. Termes liés à la confirmation et à la confiance

### Élément de confirmation

Terme générique interne désignant ce qui permet de confirmer une information ou une exécution.

Ce terme ne doit pas toujours être affiché tel quel.

Selon le contexte, utiliser :

- photo ;
- document ;
- justificatif ;
- validation ;
- constat ;
- confirmation d'intervention ;
- source.

### Photo

Image ajoutée pour montrer une situation, une intervention ou un résultat.

### Document

Fichier joint apportant une information utile.

### Justificatif

Document attendu pour confirmer une dépense, une livraison, une autorisation ou une opération formelle.

### Confirmation d'exécution

Élément permettant de confirmer qu'une action a effectivement été réalisée.

### Source

Origine d'une information.

Exemples :

- acteur terrain ;
- service public ;
- document ;
- observation ;
- système externe.

### Trace

Historique conservé par Mbàmbulaan indiquant ce qui a été fait, par qui et quand.

### Historique

Suite chronologique des événements importants liés à un objet.

### Audit

Contrôle structuré des actions, accès et modifications.

À réserver au langage interne ou institutionnel.

## 12. Termes liés au cycle de vie

### Brouillon

Information enregistrée mais pas encore envoyée.

### Soumis

Élément officiellement transmis pour traitement.

### En analyse

Élément en cours de qualification.

### À décider

Situation nécessitant une décision d'une personne habilitée.

### En coordination

Situation pour laquelle plusieurs acteurs organisent la réponse.

### En cours

Action ou engagement commencé mais non terminé.

### Bloqué

Élément qui ne peut pas progresser sans résolution d'un obstacle.

### Terminé

Action déclarée réalisée par son responsable.

### Confirmé

Action ou résultat vérifié selon la méthode prévue.

### Clôturé

Situation terminée et officiellement fermée.

### Annulé

Élément arrêté avant son aboutissement.

### Rejeté

Élément refusé après examen.

## 13. Termes liés aux initiatives

### Initiative

Ensemble organisé d'actions et de ressources destiné à répondre à un besoin priorisé.

Une initiative peut être :

- locale ;
- institutionnelle ;
- financée ;
- expérimentale ;
- multi-acteurs.

### Objectif

Résultat précis recherché par une initiative.

### Résultat

Changement directement observable après une action ou une initiative.

### Effet

Conséquence plus large ou durable découlant des résultats.

### Impact

Transformation importante et durable à laquelle Mbàmbulaan contribue.

Le terme impact ne doit pas être utilisé pour tout résultat positif.

## 14. Termes liés au financement

### Besoin de financement

Montant ou ressource nécessaire pour réaliser une initiative ou une action.

### Opportunité de financement

Possibilité identifiée de mobiliser des ressources financières.

### Engagement financier

Promesse formelle d'apporter un financement.

### Décaissement

Versement effectif d'une somme prévue.

### Financeur

Organisation qui apporte des ressources financières.

### Payeur

Acteur qui paie Mbàmbulaan pour le service.

Le financeur du projet et le payeur du produit peuvent être différents.

### Bénéficiaire économique

Acteur qui bénéficie d'un gain économique, même s'il ne paie pas directement.

## 15. Termes liés à la mesure

### Indicateur

Mesure utilisée pour suivre une activité, un résultat ou une évolution.

### Mesure

Valeur observée à une date donnée.

### Baseline

Situation de référence mesurée avant une intervention.

Dans les échanges terrain, préférer :

- « situation de départ » ;
- « point de départ ».

### Cible

Valeur ou résultat attendu.

### Seuil d'alerte

Niveau à partir duquel une attention ou une action est nécessaire.

### Résultat vérifié

Résultat confirmé par la méthode prévue.

## 16. Termes à éviter ou à encadrer

### Preuve

À éviter comme libellé générique dans les interfaces.

Raison : connotation juridique, policière ou accusatoire.

Utiliser le terme adapté au contexte.

### Ticket

À éviter pour les acteurs de terrain.

Utiliser :

- signalement ;
- demande ;
- situation.

### Workflow

À réserver au langage interne.

Utiliser :

- parcours ;
- processus ;
- étapes de traitement.

### Dashboard

À réserver au langage interne.

Utiliser :

- vue d'ensemble ;
- synthèse ;
- suivi.

### Assignee

À proscrire dans les interfaces.

Utiliser :

- responsable ;
- personne chargée de l'action.

### Statut fermé

Trop vague.

Préciser :

- clôturé ;
- annulé ;
- rejeté ;
- résolu.

### Utilisateur

Acceptable dans la documentation générique, mais préférer le rôle métier dans les interfaces.

## 17. Règles de rédaction des interfaces

Les libellés doivent :

- commencer par un verbe pour les actions ;
- être courts ;
- éviter les sigles ;
- préciser la conséquence ;
- utiliser le rôle réel de l'acteur ;
- éviter les termes administratifs non nécessaires.

Exemples :

- « Signaler un problème » ;
- « Confirmer l'intervention » ;
- « Demander un complément » ;
- « Choisir un responsable » ;
- « Indiquer la raison » ;
- « Ajouter une photo ».

## 18. Règles de traduction

Avant de traduire un terme :

- vérifier qu'il est compris en français ;
- valider son sens métier ;
- éviter la traduction littérale ;
- tester le terme avec les acteurs ;
- conserver une correspondance entre les langues.

Le wolof doit être traité comme une langue d'usage produit, pas comme une simple traduction secondaire.

## 19. Gouvernance du lexique

Chaque nouveau terme doit préciser :

- sa définition ;
- les contextes d'utilisation ;
- les termes proches ;
- les termes à éviter ;
- le libellé utilisateur ;
- le nom technique éventuel.

Les modifications importantes sont validées par :

- produit ;
- métier ;
- design ;
- terrain.

## 20. Questions ouvertes

- Quel terme utiliser localement pour « signalement » selon les territoires ?
- Faut-il distinguer « problème », « besoin » et « alerte » dès le premier écran ?
- Quel terme wolof traduit le mieux la notion de coordination organisée ?
- Le mot « engagement » est-il suffisamment compris par tous les profils ?
- Le terme « situation » est-il trop générique dans certains contextes ?
- Faut-il privilégier « responsable » ou « personne chargée de l'action » sur le terrain ?

## 21. Principe directeur

> Le vocabulaire de Mbàmbulaan doit être suffisamment simple pour être utilisé sur le terrain et suffisamment précis pour structurer durablement le produit.
