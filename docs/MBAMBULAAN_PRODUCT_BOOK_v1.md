## 1. Executive Summary

Mbàmbulaan est un écosystème numérique de coordination pour la pêche artisanale sénégalaise.

Ce n'est ni une marketplace simple, ni un ERP, ni un dashboard isolé, ni un site vitrine. La plateforme relie les arrivages, les besoins, les acteurs, les territoires, les opportunités, les transactions, les alertes, l'impact et les décisions.

Sa promesse est de rendre la filière plus lisible afin d'agir plus vite, mieux orienter les ressources, réduire les pertes d'information et mesurer la valeur créée.

| Dimension | Synthèse |
| --- | --- |
| Produit | Plateforme adaptative de coordination territoriale |
| Filière initiale | Pêche artisanale sénégalaise |
| Valeur centrale | Transformer des signaux dispersés en décisions utiles |
| Différenciation | Coordination, traçabilité, confiance, données, territoires, intelligence métier |
| Ambition | Infrastructure sectorielle réplicable à d'autres filières |

## 2. Vision

### Mission

Mbàmbulaan a pour mission de rendre la pêche artisanale sénégalaise plus coordonnée, plus lisible et plus capable de décider collectivement.

### Vision

Mbàmbulaan veut devenir l'infrastructure numérique de coordination de la pêche artisanale sénégalaise.

### Ambition à 10 ans

À 10 ans, Mbàmbulaan doit pouvoir soutenir une filière halieutique plus résiliente, mieux organisée et mieux pilotée, puis poser des fondations extensibles à d'autres filières où la coordination, la confiance et l'information de terrain sont critiques.

## 3. Les problèmes résolus

| Problème | Effet actuel | Réponse Mbàmbulaan |
| --- | --- | --- |
| Fragmentation | Chaque acteur voit une partie du système | Consolidation des signaux utiles |
| Manque de coordination | Offres, besoins et capacités ne se rencontrent pas toujours | Moteur de coordination et opportunités |
| Faible visibilité | Lots, besoins, tensions et risques sont peu lisibles | Arrivages, besoins, cartes, notifications |
| Difficulté de décision | Décisions prises avec signaux incomplets | Priorités, alertes, dashboard, executive |
| Circulation imparfaite | Information trop lente ou locale | Notifications, espaces, réseau humain |
| Manque d'intelligence collective | Les signaux ne deviennent pas action collective | Engines, analytics, decision intelligence |

## 10. Les parcours

| --- | --- |
| Terrain | Invitation -> connexion -> action utile -> opportunité -> transaction -> historique |
| Acheteur | Connexion -> besoin -> opportunités -> réservation -> transaction -> récurrence |
| Territorial | Carte -> tensions -> priorités -> décision -> impact -> reporting |
| Institutionnel | Synthèse -> risques -> arbitrage -> programme -> suivi |
| Démo | Audience -> scénario adapté -> valeur utile -> preuve contrôlée -> prochaine action |

## 24. Glossaire

| Terme | Définition |
| --- | --- |
| Arrivage | Déclaration d'un lot ou volume débarqué |
| Besoin | Demande structurée d'achat ou d'approvisionnement |
| Lot | Unité suivie depuis débarquement jusqu'à transaction ou historique |
| Opportunité | Correspondance entre un lot et un besoin |
| Transaction | Suivi d'une réservation ou mise en relation activée |
| Quai | Point territorial d'activité halieutique |
| Tension | Situation où demande, offre, risque ou capacité exigent action |
| Référent | Acteur terrain qui qualifie ou relaie les informations |
| Engine | Moteur métier indépendant de l'interface |
| Workspace | Accueil connecté contextualisé par rôle |
| Executive | Lecture institutionnelle synthétique |

| Trust | Signal de confiance lié aux acteurs |
