# Pack institutionnel Mbàmbulaan

## 1. Objet

Ce document prépare la première démarche institutionnelle de Mbàmbulaan auprès du Ministère en charge de la pêche.

L’objectif n’est pas de présenter une simple application, mais une infrastructure de coordination progressive permettant de mieux faire remonter les difficultés du terrain, organiser les réponses, suivre les interventions et produire une information utile à la décision publique.

---

# Partie A — Note de proposition de pilote

## 2. Problème public

Dans la pêche artisanale, de nombreuses difficultés opérationnelles sont connues localement mais restent mal structurées :

- informations dispersées entre appels, messages et comptes rendus ;
- absence de visibilité partagée sur les responsabilités ;
- difficulté à suivre les délais de prise en charge ;
- faible traçabilité des interventions ;
- difficultés récurrentes peu capitalisées ;
- faible visibilité institutionnelle sur les blocages réels du terrain.

Cette situation limite la capacité des acteurs à coordonner efficacement les réponses et réduit la qualité des décisions d’investissement et d’appui.

## 3. Proposition Mbàmbulaan

Mbàmbulaan propose une infrastructure numérique commune permettant de :

- faire remonter une difficulté depuis le terrain ;
- comprendre la situation ;
- identifier qui doit agir ;
- organiser une intervention ;
- suivre l’avancement ;
- constater le résultat ;
- consolider l’information pour la décision publique.

Mbàmbulaan ne se substitue pas aux services publics, aux collectivités, aux organisations professionnelles ou aux structures locales. Il leur donne un cadre commun pour mieux voir, mieux coordonner et mieux mesurer.

## 4. Hypothèse de pilote

### Périmètre

- 1 site de débarquement ;
- 1 problème prioritaire de coordination ;
- 8 semaines ;
- 4 catégories d’acteurs maximum ;
- 1 restitution finale au Ministère.

### Cas d’usage principal

Une machine à glace tombe en panne sur un site de débarquement.

La panne crée un risque de pertes post-capture et affecte plusieurs acteurs. Un représentant du site remonte la difficulté. Le coordinateur territorial comprend la situation, identifie la structure compétente, organise l’intervention et suit l’avancement. Une photo et un commentaire permettent ensuite de constater le retour en service de l’équipement.

### Acteurs du pilote

- acteur terrain ou représentant du site ;
- coordinateur territorial ;
- structure responsable de l’intervention ;
- point focal institutionnel.

## 5. Résultats attendus

Le pilote doit permettre de vérifier :

- qu’une difficulté peut être remontée simplement ;
- que les responsabilités deviennent plus lisibles ;
- que le suivi de l’intervention est compréhensible ;
- que les retards et blocages deviennent visibles ;
- que le résultat final peut être constaté ;
- que l’information consolidée devient utile au Ministère.

## 6. Indicateurs du pilote

Maximum cinq indicateurs :

1. nombre de difficultés remontées ;
2. part des situations prises en charge ;
3. délai moyen de prise en charge ;
4. part des situations réglées ;
5. nombre de blocages identifiés et levés.

## 7. Demande au Ministère

Demande principale :

> Autoriser et co-parrainer un pilote Mbàmbulaan sur un site de débarquement, désigner un point focal institutionnel et faciliter l’accès aux acteurs concernés.

Demandes complémentaires possibles :

- lettre d’intérêt ou de soutien ;
- mise en relation avec une direction technique ;
- accès à un site pilote ;
- mobilisation d’un programme ou projet partenaire ;
- financement d’amorçage du pilote ;
- mise en relation avec un bailleur pertinent.

## 8. Contrepartie pour le Ministère

Le Ministère obtient :

- un pilote limité et maîtrisé ;
- une meilleure visibilité sur les situations réelles ;
- un suivi structuré des interventions ;
- des indicateurs simples ;
- une restitution documentée ;
- une base de décision pour une éventuelle extension.

# Partie B — Script de démonstration de 10 minutes

## 9. Minute 0 à 1 — Le problème

Sur les sites de débarquement, les difficultés existent et sont connues. Mais l’information circule souvent par appels, messages, groupes informels et comptes rendus dispersés.

Le problème n’est pas seulement l’absence de données. Le problème est l’absence d’un cadre commun permettant de savoir :

- ce qui se passe ;
- où cela se passe ;
- qui doit agir ;
- où en est la réponse ;
- ce qui a réellement été réglé.

## 10. Minute 1 à 2 — Pourquoi les méthodes actuelles ne suffisent pas

Les acteurs travaillent déjà. Les institutions interviennent déjà. Les organisations locales font déjà remonter des informations.

Mais sans infrastructure commune :

- certaines situations se perdent ;
- les responsabilités sont floues ;
- les retards sont difficiles à suivre ;
- les mêmes problèmes se répètent sans capitalisation ;
- la décision publique manque de visibilité opérationnelle.

## 11. Minute 2 à 5 — Démonstration du parcours

### Écran 1 — Acteur terrain

Question affichée :

> Que voulez-vous faire aujourd’hui ?

L’acteur choisit :

> Remonter une difficulté.

Il indique :

- ce qui se passe ;
- où cela se passe ;
- depuis quand ;
- qui est concerné ;
- si c’est urgent ;
- une photo ou un document.

### Écran 2 — Situation créée

La situation apparaît avec un statut simple :

> Reçu.

L’acteur peut ensuite voir :

- qui s’en occupe ;
- où en est la situation ;
- quelle est la prochaine étape.

### Écran 3 — Coordinateur

Question affichée :

> Qu’est-ce qui demande votre attention ?

Le coordinateur voit :

- les difficultés à examiner ;
- les situations sans responsable ;
- les interventions à organiser ;
- les retards ;
- les situations réglées.

Il ouvre la panne de la machine à glace, identifie la structure responsable et organise l’intervention.

### Écran 4 — Suivi

Les statuts évoluent :

- Reçu ;
- En cours d’examen ;
- Pris en charge ;
- Intervention prévue ;
- Intervention en cours ;
- Réglé.

L’acteur terrain et le coordinateur voient la même situation, avec un niveau d’information adapté à leur rôle.

### Écran 5 — Résultat

Une photo et un commentaire montrent que la machine fonctionne à nouveau.

Le résultat devient visible, exploitable et capitalisable.

## 12. Minute 5 à 7 — Ce que voit le Ministère

Le Ministère peut voir :

- les sites touchés ;
- les difficultés prioritaires ;
- les situations sans réponse ;
- les retards ;
- les interventions engagées ;
- les résultats obtenus.

Le Ministère ne reçoit pas seulement plus d’informations. Il obtient une meilleure capacité de pilotage.

## 13. Minute 7 à 9 — Le pilote

Nous proposons un pilote limité :

- un site ;
- un cas d’usage prioritaire ;
- huit semaines ;
- un point focal ;
- une restitution finale.

L’objectif n’est pas de promettre un déploiement national immédiat. L’objectif est de vérifier rapidement la valeur opérationnelle, la capacité de coordination et l’utilité institutionnelle.

## 14. Minute 9 à 10 — La demande

> Nous sollicitons l’accord du Ministère pour co-construire ce pilote, désigner un point focal et faciliter l’accès au site et aux acteurs concernés.

Nous proposons ensuite une restitution documentée permettant de décider, sur la base de résultats, s’il faut poursuivre, adapter ou élargir le dispositif.

# Partie C — Dossier d’aide à la décision

## 15. Décision attendue

Décision minimale recherchée :

- accord pour approfondir techniquement ;
- désignation d’un point focal ;
- identification d’un site pilote ;
- autorisation de co-construction du pilote.

Décision renforcée possible :

- lettre de soutien ;
- financement du pilote ;
- rattachement à un programme existant ;
- mobilisation d’un partenaire technique ou financier.

## 16. Pourquoi maintenant

La pêche artisanale repose sur de nombreux acteurs interdépendants. Les difficultés liées aux équipements, aux services, à la qualité, à la sécurité et à l’organisation locale ont des effets directs sur la conservation, les revenus et la continuité des activités.

Une infrastructure de coordination progressive permet de commencer petit, produire des résultats rapides et améliorer la décision sans attendre une réforme lourde ou un système national complet.

## 17. Risques du statu quo

- pertes économiques liées aux délais d’intervention ;
- répétition des mêmes dysfonctionnements ;
- faible traçabilité des réponses ;
- faible capitalisation des besoins ;
- difficulté à prioriser les investissements ;
- perte de confiance entre terrain et institutions.

## 18. Risques du pilote

### Risque : faible adoption terrain

Réponse : parcours simple, nombre limité d’acteurs, accompagnement local, vocabulaire métier.

### Risque : perception comme outil de contrôle

Réponse : positionner Mbàmbulaan comme infrastructure de coordination et non comme outil de surveillance.

### Risque : doublon avec des dispositifs existants

Réponse : intégrer les acteurs et circuits existants, ne pas les contourner.

### Risque : promesse excessive

Réponse : limiter le pilote, les indicateurs et les engagements.

### Risque : dépendance au financement public

Réponse : utiliser l’appui institutionnel comme levier de démarrage, tout en construisant un modèle économique multi-payeurs.

## 19. Conditions de réussite

- un sponsor institutionnel ;
- un point focal opérationnel ;
- un site accessible ;
- un cas d’usage prioritaire ;
- un responsable local ;
- des données limitées et fiables ;
- une restitution finale structurée.

# Partie D — Modèle économique initial

## 20. Principe

Les acteurs terrain ne doivent pas être la principale source de revenus au démarrage.

Le modèle doit reposer sur les acteurs qui retirent une valeur budgétaire, opérationnelle, réglementaire ou décisionnelle de la coordination.

## 21. Payeurs potentiels

### Institutions publiques

Motifs de paiement :

- pilotage territorial ;
- suivi de programmes ;
- gestion des infrastructures ;
- visibilité sur les interventions ;
- indicateurs d’impact.

Formes de revenus :

- contrat de service ;
- abonnement institutionnel ;
- financement de pilote ;
- marché public ;
- convention de partenariat.

### Programmes et bailleurs

Motifs de paiement :

- suivi des activités financées ;
- traçabilité des résultats ;
- mesure d’impact ;
- coordination multi-acteurs.

Formes de revenus :

- licence projet ;
- prestation de déploiement ;
- suivi-évaluation numérique ;
- accompagnement opérationnel.

### Collectivités et structures territoriales

Motifs de paiement :

- suivi local des besoins ;
- coordination des services ;
- priorisation des interventions.

Formes de revenus :

- abonnement territoire ;
- contrat annuel ;
- mutualisation intercommunale.

### Organisations professionnelles et opérateurs privés

Motifs de paiement :

- réduction des pertes ;
- meilleure continuité des opérations ;
- visibilité sur les équipements et services ;
- fiabilisation des relations entre acteurs.

Formes de revenus :

- modules métier ;
- abonnement organisation ;
- services premium ;
- intégration et accompagnement.

## 22. Logique de démarrage

Phase 1 — financement pilote :

- subvention ;
- partenariat institutionnel ;
- programme d’appui ;
- financement bailleur ;
- contribution en nature.

Phase 2 — revenus récurrents :

- abonnements institutionnels ;
- contrats de suivi de programmes ;
- licences territoriales ;
- accompagnement et déploiement.

Phase 3 — extension de l’écosystème :

- modules métier ;
- transactions encadrées ;
- services de données ;
- intégrations ;
- services financiers ou logistiques via partenaires, uniquement lorsque la coordination de base est validée.

## 23. Ce qu’il ne faut pas faire

- faire payer les pêcheurs dès le départ ;
- dépendre exclusivement d’un Ministère ;
- baser le modèle sur la publicité ;
- lancer trop tôt une marketplace ;
- monétiser des données sensibles sans cadre clair ;
- promettre une rentabilité avant d’avoir validé le moteur de valeur.

# Partie E — Stratégie d’adoption et de légitimité

## 24. Objectif

Construire simultanément :

- légitimité institutionnelle ;
- utilité terrain ;
- crédibilité opérationnelle ;
- capacité de financement.

## 25. Séquence recommandée

1. obtenir une première porte d’entrée institutionnelle ;
2. sécuriser un point focal ;
3. identifier un site crédible ;
4. conduire des entretiens terrain ;
5. adapter le parcours produit ;
6. lancer un pilote limité ;
7. mesurer ;
8. restituer ;
9. obtenir un deuxième engagement ;
10. élargir progressivement.

## 26. Sources de légitimité

- soutien d’une direction technique ;
- participation d’un CLPA ou d’une structure locale compétente ;
- partenariat avec une organisation professionnelle ;
- implication d’un acteur terrain reconnu ;
- résultats documentés ;
- gouvernance claire ;
- transparence sur les limites du pilote.

## 27. Critères de passage à l’échelle

Ne pas élargir tant que les conditions suivantes ne sont pas réunies :

- les acteurs comprennent la valeur ;
- le parcours est utilisé sans forte assistance ;
- les responsabilités deviennent plus lisibles ;
- les données produites sont réellement exploitées ;
- au moins un payeur potentiel confirme son intérêt ;
- le coût de déploiement reste compatible avec la valeur créée.
