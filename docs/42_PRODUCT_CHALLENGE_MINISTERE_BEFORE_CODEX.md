# Product challenge - Ministere avant prochaine passe Codex

## Decision

Oui, il faut encore challenger le produit avant de relancer Codex.

Pas pour ajouter des idees. Pour eviter que Mbambulaan devienne une interface jolie mais pas encore achetable.

La prochaine passe doit prouver une promesse centrale : le Ministere peut passer d'une lecture nationale a une action locale documentee, quai par quai.

## Question centrale

Pourquoi le Ministere paierait pour Mbambulaan au lieu de continuer avec Excel, WhatsApp, rapports PDF, reunions et tableaux de bord classiques ?

Reponse attendue : parce que Mbambulaan relie territoire, quai, programme, budget, ressource, incident, preuve, referent terrain, IA et action dans un meme parcours.

## Promesse a prouver

Vision nationale -> choix du quai -> KPIs dynamiques -> insight IA -> referents terrain -> action -> trace.

Si cette chaine n'est pas visible, le produit reste trop proche d'un dashboard.

## Angles a challenger

### 1. La carte est-elle vraiment une porte d'entree ?

Une carte statique ne suffit pas.

La carte doit permettre de choisir un quai et de faire changer toute la lecture : KPIs, budget, incidents, ressources, referents, preuves, IA et actions.

### 2. Les KPIs sont-ils decisionnels ?

Un KPI qui ne mene pas a une action est faible.

Chaque KPI doit repondre a une question :

- pourquoi ce quai est prioritaire ?
- quel risque domine ?
- quel financement bloque ?
- quelle ressource manque ?
- quel referent contacter ?
- quelle preuve manque ?

### 3. Les referents terrain creent-ils une vraie coordination ?

Les referents ne sont pas des contacts CRM.

Ils doivent permettre :

- verification terrain ;
- compte rendu ;
- remontée besoin ;
- suivi programme ;
- appui financement ;
- alerte incident ;
- lien entre Ministere et acteurs locaux.

### 4. Les actions vont-elles jusqu'au bout ?

Un simple message ne suffit pas.

Chaque action doit laisser une trace visible : statut modifie, action en attente, brouillon de note, preuve demandee, responsable assigne.

### 5. L'IA est-elle utile ou decorative ?

L'IA doit produire un insight court et contextualise au quai selectionne.

Elle doit dire :

- pourquoi ce quai est prioritaire ;
- ce qui bloque ;
- quelle action faire ;
- quelle donnee manque ;
- quelle note preparer.

L'IA assiste, l'humain valide.

### 6. Le design fait-il analytics premium ?

Les blocs de texte et cartes simples doivent etre limites.

Priorite a :

- donuts ;
- progress bars ;
- gauges ;
- mini charts ;
- score radial ;
- badges ;
- tableaux compacts ;
- palette mer.

### 7. Le Ministere comprend-il en moins de 10 secondes ?

Chaque vue doit afficher :

- ce qu'il faut regarder ;
- pourquoi c'est important ;
- quelle action faire.

## Risques produit

- ajouter du joli sans valeur metier ;
- ajouter trop de KPIs sans action ;
- rendre les referents trop CRM ;
- faire une BI generique ;
- faire un ERP masque ;
- perdre la logique de coordination ;
- montrer une seule zone prioritaire au lieu d'un vrai filtre dynamique ;
- ne pas faire reagir l'interface au choix du quai.

## Definition d'un MVP Ministere convaincant

Le MVP est convaincant si un utilisateur peut :

1. ouvrir l'espace Ministere ;
2. comprendre la situation nationale ;
3. selectionner Joal, Kayar, Mbour, Saint-Louis ou Dakar ;
4. voir les KPIs changer ;
5. comprendre le risque dominant ;
6. identifier les referents utiles ;
7. declencher une action ;
8. voir une trace ;
9. generer une note ou insight IA ;
10. expliquer pourquoi Mbambulaan facilite la decision.
