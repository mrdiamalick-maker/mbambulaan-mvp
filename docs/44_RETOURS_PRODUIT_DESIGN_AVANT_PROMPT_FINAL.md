# Retours produit et design avant prompt final

## Decision

Avant de redonner un prompt a Codex, il faut aligner la direction produit et design. La prochaine passe doit etre la passe de finition pilote MVP pour presentation Ministere.

Priorite : landing et espace prive Etat / Ministere.

## Retours landing

La landing doit devenir beaucoup plus premium et plus vendeuse.

Problemes observes :

- rendu encore trop simple ;
- certains blocs ressemblent a des encadres PowerPoint ;
- wording pas assez accrocheur ;
- cas d'usage trop simplistes ;
- explication solution trop basique ;
- manque de pictos, figures, workflow et mise en scene visuelle ;
- couleurs mer pas assez presentes.

Direction attendue :

- experience B2B premium ;
- palette mer plus assumee ;
- wording vendeur et clair ;
- CTA simples, beaux et evidents ;
- blocs mieux chartes ;
- pictos et figures propres ;
- workflow visuel pour expliquer la solution ;
- donner envie de decouvrir et demander une demo.

Message landing a renforcer :

Mbambulaan transforme les signaux terrain disperses en decisions coordonnees, suivies et tracables pour la filiere peche artisanale.

## Retours espace Ministere

L'espace Ministere est sur la bonne voie, mais doit etre rendu plus premium.

Problemes observes :

- composants trop resserres horizontalement ;
- manque de respiration verticale ;
- les lignes de KPIs et composants doivent mieux utiliser la palette mer ;
- synthese, territoires, tableaux, cartes, actions, referents doivent etre mieux designes ;
- l'IA est trop systematiquement presente dans les blocs, ce qui induit en erreur ;
- il faut valoriser l'IA mais la rendre activable/desactivable comme prevu.

Direction attendue :

- accepter le scroll vertical ;
- mieux separer les sections ;
- rendre les KPIs plus premium ;
- rendre les cartes, tableaux, actions, referents et fiches plus soignes ;
- travailler la palette mer partout ;
- rendre chaque action plus complete ;
- aller au bout de chaque parcours ;
- rendre l'IA visible comme capacite optionnelle et gouvernee.

## IA Mbambulaan

L'IA ne doit pas etre partout par defaut.

Elle doit etre :

- valorisee ;
- activable/desactivable ;
- gouvernee ;
- rassurante ;
- clairement marquee simulation/donnees mockees ;
- presente comme aide a la synthese, a l'analyse et a la preparation de notes ;
- jamais comme decision automatique.

Proposition UX :

- un panneau ou module IA dedie ;
- toggles par capacite : Synthese, Alertes, Notes, Recherche assistee ;
- quand l'IA est activee, elle enrichit les sections ;
- quand elle est desactivee, les sections restent fonctionnelles sans IA.

## Regle produit

Ne pas ajouter du joli sans valeur metier.

Le rendu doit etre premium parce qu'il rend la decision plus claire, pas parce qu'il ajoute de la decoration.

## Objectif du prochain prompt Codex

En un seul prompt, demander une passe finale de finition :

- landing premium, vendeuse, mieux chartee ;
- espace Ministere plus premium, plus vertical, plus lisible ;
- palette mer appliquee aux KPIs, cartes, tableaux, actions, referents ;
- IA activable/desactivable ;
- actions et parcours menes jusqu'au bout ;
- P2 uniquement si P1 solide.

## Definition de succes

La version est aboutie pour la phase pilote MVP si :

1. la landing donne envie de demander une demo ;
2. le Ministere se reconnait dans le discours ;
3. l'espace Ministere impressionne sans devenir gadget ;
4. le filtre par quai est central ;
5. les actions produisent une trace visible ;
6. l'IA est valorisee sans etre envahissante ;
7. le design semble construit, pas assemble ;
8. typecheck et build passent.
