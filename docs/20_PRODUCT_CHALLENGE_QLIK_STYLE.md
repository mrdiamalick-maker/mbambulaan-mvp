# Product challenge - Qlik style pour Mbambulaan

## Constat

La version actuelle progresse. Mais les espaces prives restent trop proches d'une visualisation unique. Les menus de gauche doivent devenir de vrais modules metier. Un clic sur un menu doit afficher une vue differente, avec ses composants, ses donnees et ses actions.

Mbambulaan vend une solution privee. L'espace ne doit donc pas seulement montrer des indicateurs. Il doit aider chaque acteur a travailler, decider, coordonner et prouver.

## Inspiration

S'inspirer de Qlik pour la logique visuelle et produit : analytics, filtres, tableaux de bord, reporting, automatisation, passage de la donnee vers l'action.

Ne pas copier Qlik. Adapter cette logique a Mbambulaan : quais, acteurs, signaux, flux, preuves, territoires, programmes, financements et decisions.

## Direction design

Reduire le noir dominant.

Palette recommandee :

- bleu ocean profond ;
- turquoise pour actions et highlights ;
- sable clair pour fonds secondaires ;
- vert lagon pour statuts positifs ;
- corail ou ambre pour alertes.

Les boutons doivent toujours etre lisibles.

## KPIs

Les KPIs doivent ressembler davantage a Power BI, Qlik ou Looker Studio : carte compacte, tendance, mini-viz, contexte, insight et clic possible pour filtrer.

Un KPI doit aider a decider, pas seulement afficher un chiffre.

## Carte

La carte devient un vrai module dedie, surtout pour le Ministere.

Attendu : grande cartographie stylisee, points de quais, acteurs, tensions, programmes, financement, preuves, panneau detail au clic et filtres.

Prevoir une structure de donnees compatible avec latitude et longitude pour plus tard.

## Menus dynamiques

Chaque acteur doit avoir ses propres menus.

Ministere : Synthese, Carte des quais, Territoires, Tensions, Programmes, Financements, Notes, Preuves, Admin.

ONG : Vue programme, Zones, Beneficiaires, Activites, Preuves, Risques, Reporting, Admin programme.

Collectivite : Commune, Carte locale, Quais, Demandes, Actions, Partenaires, Note mairie, Admin local.

Pecheur : Mon statut, Signaler, Relais, Mes demandes, Retours.

Mareyeur : Flux du jour, Lots, Qualite, Froid, Retraits, Risques, Historique.

Exportateur : Supply qualifie, Opportunites privees, Preuves, Risques, Conditions, Decision memo.

Organisation : Bureau, Membres, Demandes collectives, Carte membres, Dossiers partenaires, Plaidoyer, Admin organisation.

Investisseur : These, Segments payeurs, Offres, Pipeline, Carte expansion, Revenus, Roadmap, Risques, Data room.

## Actions qui justifient le prix

Chaque espace doit permettre au minimum : filtrer, ouvrir un detail, changer un statut, affecter ou prioriser une action, ajouter une note, generer une synthese, preparer un rapport, simuler une validation.

Si le backend n'existe pas encore, l'action doit fonctionner cote client avec une confirmation claire.

## Admin prive

Ajouter une logique admin simulee : roles, permissions, modules actives, historique d'action, statut brouillon / valide / archive.

Tous les profils n'ont pas le meme niveau d'administration.

## Landing

La landing doit etre plus proche d'une solution B2B data et coordination : promesse claire, cas d'usage, preuve produit, parcours par acteur, CTA demo qualifiee et devis.

## Critere CEO

Le CEO doit pouvoir montrer a son associe que Mbambulaan est une couche de coordination vendable, avec des espaces metier, une carte des quais, des actions utiles et une valeur claire par payeur.
