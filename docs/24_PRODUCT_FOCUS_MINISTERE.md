# Focus produit - Espace Ministere

## Diagnostic CPO

La solution progresse, mais il faut maintenant arreter de tout traiter en meme temps. On doit prendre un espace, le rendre excellent, puis reutiliser la logique pour les autres acteurs.

Premier espace prioritaire : Ministere.

Si le Ministere accede a Mbambulaan, il ne doit pas voir une interface admin. Il doit voir une solution qui l'aide a maitriser la peche artisanale : suivi, cartographie, alertes, programmes, financement, conflits, preuves, notes et aide a la decision.

## Positionnement

Ne plus utiliser le mot cockpit.

Nom recommande : `Espace de pilotage institutionnel`.

Variantes possibles :

- Centre de coordination institutionnelle ;
- Vue de decision nationale ;
- Tableau de bord institutionnel ;
- Espace Ministere.

## Probleme reel du Ministere

Hypotheses fortes :

- donnees fragmentees ;
- fichiers Excel disperses ;
- donnees parfois obsoletes ;
- peu de vision temps reel des quais ;
- difficulte a suivre les programmes ;
- doublons entre projets ;
- tensions locales mal remontees ;
- manque de cartographie operationnelle ;
- reporting lent ;
- decision dependante de notes manuelles.

Mbambulaan doit repondre a ces douleurs.

## Promesse Ministere

Mbambulaan aide le Ministere a voir, comprendre, prioriser, coordonner et documenter l'action publique sur la peche artisanale.

Valeur vendable :

- maitrise territoriale ;
- meilleure priorisation ;
- suivi des programmes ;
- reduction des doublons ;
- meilleure gestion des tensions ;
- rapports plus rapides ;
- notes de decision assistees ;
- cartographie des quais et acteurs ;
- base de preuves exploitable.

## Modules attendus

### Synthese nationale

Objectif : comprendre la situation en 60 secondes.

Afficher :

- indicateurs cles ;
- zones critiques ;
- decisions attendues ;
- alertes majeures ;
- programmes a risque ;
- synthese IA simulee.

Actions : generer synthese, marquer decision, ouvrir zone critique.

### Carte des quais

Objectif : visualiser les quais et acteurs.

Afficher :

- grande carte ;
- points avec lat/lng mockes ;
- tension par couleur ;
- acteurs presents ;
- signaux recents ;
- programmes actifs ;
- financements ;
- panneau detail.

Actions : prioriser zone, demander verification, generer note, ouvrir fiche territoire.

### Territoires

Objectif : comparer et prioriser les zones.

Afficher : classement, score priorite, tension, volume, acteurs, preuve, action recommandee.

Actions : classer priorite, affecter suivi, exporter liste.

### Tensions et alertes

Objectif : suivre conflits, risques, urgences et signaux sensibles.

Afficher : alertes, severite, territoire, responsable, statut, delai.

Actions : changer statut, assigner responsable, creer note d'alerte.

### Programmes

Objectif : suivre les interventions publiques et partenaires.

Afficher : portefeuille programmes, territoire, budget, avancement, partenaire, risque de doublon.

Actions : detecter doublon, ouvrir programme, marquer risque, generer point suivi.

### Financements

Objectif : suivre les demandes, priorites et montants.

Afficher : dossiers, montants, urgence, preuve, programme associe, statut.

Actions : prioriser dossier, demander preuve, preparer arbitrage.

### Acteurs

Objectif : voir qui agit ou influence chaque zone.

Afficher : services, communes, ONG, organisations, relais, mareyeurs, exportateurs.

Actions : ouvrir fiche, filtrer par territoire, affecter contact.

### Notes et rapports

Objectif : gagner du temps sur la production de notes.

Afficher : notes brouillon, notes pretes, rapports, destinataire, statut.

Actions : generer note IA simulee, passer en validation, archiver.

### Preuves

Objectif : savoir ce qui est fiable.

Afficher : source, niveau de preuve, territoire, date, validation.

Actions : demander verification terrain, valider preuve, relier preuve a decision.

### Parametres et acces

Objectif : gerer l'espace achete.

Afficher : utilisateurs, roles, permissions, modules actives, historique.

Actions : modifier role, activer module, consulter historique.

## IA Mbambulaan pour Ministere

Ajouter une logique IA simulee visible :

- synthese automatique de situation ;
- detection de doublon ;
- recommandation de priorite ;
- preparation de note ;
- resume territoire ;
- alerte sur donnees obsoletes ;
- questions rapides : Que se passe-t-il a Joal ? Quels programmes sont a risque ? Quelles zones prioriser ?

Ne pas surpromettre. Afficher `IA simulee - donnees mockees` si necessaire.

## Bloc apercu solution

Si l'apercu n'est pas explicable, le remplacer.

Nouvelle idee : un flow simple en 4 etapes :

Signal terrain -> Analyse territoriale -> Coordination -> Decision / Rapport.

Ou une visualisation claire : carte + alertes + note generee.

## Cas d'usage landing

Remplacer les cartes banales par un flow ou une matrice simple :

Acteurs -> Besoins -> Capacites Mbambulaan -> Valeur.

Exemple : Ministere -> arbitrer -> carte + alertes + notes -> decision plus rapide.

## Regle de focus

Ne pas retravailler tous les espaces maintenant.

Priorite unique : rendre l'espace Ministere convaincant, clair, premium et achetable.

Quand le Ministere sera solide, on appliquera la meme exigence a l'ONG, puis au mareyeur, etc.
