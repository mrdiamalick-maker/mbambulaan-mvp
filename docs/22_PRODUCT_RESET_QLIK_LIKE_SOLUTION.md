# Product reset - Mbambulaan solution B2B

## Diagnostic

La version actuelle progresse, mais elle ressemble encore trop a un site avec pages admin.

Mbambulaan doit etre percu comme une solution B2B : integration, analyse, geolocalisation, coordination, reporting et action.

La reference d'inspiration est Qlik pour la logique produit : une facade simple, une plateforme puissante, des donnees qui menent a des decisions.

## Landing

La landing doit etre simple, epuree, claire.

Le header doit etre retravaille : navigation plus lisible, CTA visibles, distinction prospect et partenaire connecte.

Le prospect doit pouvoir demander un essai ou une demo. Le partenaire acquis doit se connecter.

Ne pas afficher Mbambulaan comme un site public de modules.

Structure attendue :

- hero tres clair ;
- promesse ;
- problemes de coordination ;
- ce que la solution connecte ;
- cas d'usage par acteur ;
- apercu produit lisible ;
- CTA demander un essai, demander une demo, se connecter.

## Terminologie

Eviter le mot cockpit.

Utiliser plutot : espace de pilotage, centre de coordination, vue de decision, espace programme, espace partenaire, tableau de bord institutionnel.

## Demo

`/demo` et `/demo/<role>` doivent etre repenses.

La demo doit qualifier le prospect : qui est-il, quel probleme veut-il resoudre, quelle valeur veut-il voir.

Les pages role doivent montrer une valeur en quelques etapes puis pousser vers demande d'essai ou devis.

## Demande demo

`/demande-demo` devient une demande d'essai qualifiee.

Champs attendus : organisation, profil, territoire, probleme principal, acteurs concernes, urgence, besoin d'integration, contact.

## Espace prive

`/espace-prive` doit ressembler a une entree partenaire, pas a un portail public.

Les espaces prives doivent etre plus simples, plus utiles, plus actionnables.

Chaque menu doit afficher un module different avec titre, KPIs, filtres, tableau ou carte, detail, actions, synthese.

## Carte

La cartographie doit devenir un module dedie.

Attendu : grande carte, points de quais, lat/lng mockes, filtres, details au clic, acteurs, tension, programmes, financement, preuves et actions.

## KPIs

Les KPIs doivent etre plus analytics : valeur, tendance, contexte, mini visualisation, insight et filtre au clic.

## Admin

Ajouter une logique admin simulee pour les espaces payeurs : utilisateurs, roles, permissions, modules actifs, publication, validation, historique, brouillon, publie, archive.

## Parcours cible

Visiteur public : landing vers demande d'essai.

Prospect qualifie : demo role vers demande d'essai ou devis.

Partenaire acquis : connexion vers espace prive.

Admin partenaire : espace prive vers parametres, utilisateurs et publications.

## Regles

Mbambulaan n'est pas un site public.
Mbambulaan n'est pas une marketplace.
Mbambulaan n'est pas un dashboard admin generique.
Mbambulaan est une plateforme de coordination et d'intelligence territoriale.

Le design doit etre simple, moderne, clair et inspire des plateformes data B2B.

## Validation CEO

Le CEO doit pouvoir dire : Mbambulaan integre les donnees terrain, cartographie les quais et les acteurs, analyse les signaux, coordonne les actions et produit des rapports utiles a chaque partie prenante.
