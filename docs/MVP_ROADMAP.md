# Roadmap MVP — Mbàmbulaan

## Principe de pilotage

Le projet n’avance plus par petits sprints isolés. Il avance par packs de développement.

Un pack doit livrer une capacité métier complète, avec validation technique et mise à jour de la Pull Request.

## Phase 1 — MVP fonctionnel

Objectif : démontrer un parcours métier complet.

Statut : quasiment terminé.

Capacités livrées :

- Landing page
- Arrivages
- Déclaration d’arrivage
- Besoins
- Publication de besoin
- Matching automatique
- Opportunités
- Réservation
- Transactions
- Notifications
- Dashboard
- Carte des quais
- Centre de coordination
- Espaces métiers
- Démo guidée

## Phase 2 — Intelligence métier

Objectif : faire comprendre pourquoi Mbàmbulaan crée de la valeur.

Capacités livrées ou en cours :

- Moteur de recommandation
- Référentiel métier Sénégal
- Score de confiance des acteurs
- Indicateurs d’impact
- Tension territoriale
- Priorisation métier
- Alertes intelligentes
- Traçabilité des lots

## Phase 3 — MVP Premium

Objectif : rendre la démonstration désirable et mémorable.

Capacités prévues :

- Données plus réalistes
- Amélioration UX/UI
- Harmonisation visuelle
- Micro-interactions
- Storytelling de démonstration
- Indicateurs exécutifs

## Phase 4 — Plateforme nationale

Objectif : préparer l’ambition long terme.

Axes prévus :

- Comptes utilisateurs
- Authentification
- API
- Base de données
- Administration
- Données réelles
- Reporting institutionnel
- Analytics avancés
- Intégrations partenaires

## Règle de priorité

Toute nouvelle évolution doit répondre à au moins une question :

- Quelle valeur métier est créée ?
- Quel acteur est aidé ?
- Quelle donnée est enrichie ?
- Quel module est impacté ?
