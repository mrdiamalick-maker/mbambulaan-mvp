# Catalogue de services et modules v1 - Mbambulaan

## Objectif

Donner a Mbambulaan une structure commerciale et produit claire.

Le partenaire n'achete pas une page. Il achete un socle de solution et peut activer des modules selon ses besoins.

## Niveaux d'offre

### 1. Socle standard

Pour demarrer rapidement.

Inclut :

- espace prive partenaire ;
- roles de base ;
- tableau de bord de synthese ;
- carte simple ;
- territoires ;
- acteurs ;
- alertes simples ;
- notes et rapports simples ;
- exports basiques.

### 2. Pack institutionnel

Pour Ministere, direction, programme public ou partenaire structurant.

Inclut :

- cartographie avancee ;
- suivi programmes ;
- budgets et financements ;
- ressources et infrastructures ;
- incidents et obsolescence ;
- preuves et validations ;
- IA assistee ;
- rapports institutionnels ;
- permissions avancees.

### 3. Sur mesure et integrations

Pour besoins specifiques.

Inclut selon cadrage :

- connecteurs donnees ;
- reprise fichiers Excel ;
- integration API ;
- module metier specifique ;
- deploiement dedie ;
- accompagnement terrain ;
- reporting personnalise.

## Catalogue modules

### Coordination territoriale

Probleme resolu : manque de vision coordonnee des territoires.

Valeur : prioriser les zones, suivre les actions, eviter les doublons.

Disponibilite : standard.

### Cartographie et geolocalisation

Probleme resolu : absence de lecture spatiale des quais, acteurs et tensions.

Valeur : visualiser, filtrer, comprendre, agir par territoire.

Disponibilite : standard puis avance en option.

### Suivi programmes

Probleme resolu : programmes disperses, avancement peu lisible, risque de chevauchement.

Valeur : suivre budget, avancement, partenaire, indicateurs, risques.

Disponibilite : institutionnel.

### Budgets et financements

Probleme resolu : faible visibilite sur budget prevu, engage, consomme, retards et ecarts.

Valeur : alerter, arbitrer, demander preuve, suivre execution.

Disponibilite : institutionnel.

### Ressources et infrastructures

Probleme resolu : ressources humaines, materiel et infrastructures mal suivis.

Valeur : voir disponibilite, etat, responsable, localisation, criticite.

Disponibilite : institutionnel.

### Incidents et obsolescence

Probleme resolu : incidents, pannes et obsolescence mal historises.

Valeur : signaler, suivre, prioriser, relier a financement ou maintenance.

Disponibilite : institutionnel.

### Acteurs et organisations

Probleme resolu : acteurs disperses et roles peu clairs.

Valeur : cartographier services, communes, ONG, organisations, relais, operateurs.

Disponibilite : standard.

### Preuves et validation terrain

Probleme resolu : decisions basees sur donnees non verifiees ou rapports lents.

Valeur : relier chaque donnee a une source, une date, un niveau de confiance.

Disponibilite : institutionnel.

### Notes, rapports et exports

Probleme resolu : production manuelle de notes et rapports.

Valeur : preparer notes, rapports, exports et points de suivi.

Disponibilite : standard puis avance en option.

### IA Mbambulaan

Probleme resolu : trop de donnees dispersees, analyse lente, priorisation difficile.

Valeur : synthese, questions rapides, detection doublon, alerte ecart, preparation note.

Disponibilite : optionnel / institutionnel.

Regle : l'IA assiste, l'humain decide.

### Administration, roles et permissions

Probleme resolu : controle insuffisant des acces et publications.

Valeur : roles, permissions, modules actifs, historique, validation.

Disponibilite : standard puis avance en option.

### Integrations et donnees externes

Probleme resolu : donnees dans fichiers, outils existants ou systemes partenaires.

Valeur : importer, connecter, nettoyer, structurer et exploiter.

Disponibilite : sur mesure.

## Sensibilite donnees

Chaque module doit avoir un niveau de sensibilite :

- faible : donnees publiques ou non sensibles ;
- moyen : donnees operationnelles internes ;
- eleve : budgets, incidents, notes, utilisateurs, donnees institutionnelles.

Les modules sensibles doivent avoir permissions, historique et validation.

## Regle commerciale

Ne pas vendre tout a tout le monde.

Vendre le bon socle, puis activer les bons modules selon le niveau de maturite, le budget et le besoin du partenaire.
