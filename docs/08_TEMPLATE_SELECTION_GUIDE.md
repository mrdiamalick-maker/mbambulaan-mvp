# Mbàmbulaan Template Selection Guide

## Statut du document

Ce guide définit le type de template frontend à rechercher pour reconstruire Mbàmbulaan proprement après validation du Product UX Blueprint Final.

Objectif : éviter d'acheter un template inadapté, trop marketing, trop marketplace ou trop léger pour une infrastructure de coordination.

## 1. Objectif Du Template

Le template doit servir une plateforme B2B premium, dense et opérationnelle.

Il doit permettre de construire :

- une landing premium courte ;
- un sélecteur de rôle ;
- des espaces de démo différenciés ;
- un espace premium simulé ;
- des tableaux de bord par rôle ;
- des cartes opérationnelles ;
- des tableaux de données ;
- des rapports ;
- des formulaires de cadrage ;
- des modules privés.

## 2. Types De Templates À Rechercher

Priorité haute :

- command center ;
- operations dashboard ;
- logistics dashboard ;
- data intelligence platform ;
- B2B SaaS analytics dashboard ;
- admin dashboard premium ;
- CRM opérationnel ;
- enterprise platform dashboard.

Priorité moyenne :

- project management dashboard ;
- field operations dashboard ;
- business intelligence dashboard ;
- fintech risk dashboard ;
- supply chain dashboard.

À éviter :

- marketplace ;
- e-commerce ;
- portfolio ;
- landing page SaaS simple ;
- template mobile-only ;
- template trop illustratif ;
- template trop startup grand public ;
- template avec peu de composants métier.

## 3. Critères Obligatoires

Un template candidat doit posséder :

- Next.js ou React compatible ;
- TypeScript ou structure facilement typable ;
- Tailwind CSS ou design tokens exploitables ;
- sidebar premium ;
- navigation claire ;
- dashboards denses ;
- cards KPI ;
- tables avancées ;
- badges et statuts ;
- charts simples ;
- timelines ;
- formulaires ;
- pages settings ;
- responsive desktop et mobile ;
- composants réutilisables ;
- bonne hiérarchie typographique ;
- design sobre, professionnel et institutionnel.

## 4. Critères Très Importants

Le template est fortement préféré s'il propose :

- map-like components ou zones géographiques stylisées ;
- dark panels ;
- command palette ou quick actions ;
- role-based layouts ;
- data tables avec filtres ;
- report cards ;
- activity feed ;
- notification center ;
- multi-column dashboard ;
- composants de stepper ou workflow ;
- charts horizontaux simples ;
- empty states professionnels.

## 5. Critères Éliminatoires

Rejeter un template si :

- il ressemble à une marketplace ;
- il met l'accent sur les produits à vendre plutôt que les décisions ;
- il est dominé par une landing marketing ;
- il n'a pas de vraies tables ;
- il n'a pas de dashboard dense ;
- il n'a pas de sidebar ou navigation privée crédible ;
- il est trop coloré ou gadget ;
- il dépend de trop nombreuses librairies lourdes ;
- il impose une identité visuelle impossible à adapter ;
- il ne gère pas correctement le responsive ;
- il manque de composants pour rapports, statuts et actions.

## 7. Grille De Scoring

Score sur 100.

| Critère | Pondération |
| --- | ---: |
| Compatibilité Next.js / React / TypeScript | 10 |
| Qualité du dashboard | 15 |
| Sidebar et navigation privée | 10 |
| Tables, charts, cards et badges | 15 |
| Capacité à créer `/espace-prive` | 15 |
| Sobriété premium institutionnelle | 10 |
| Responsive solide | 10 |
| Facilité de personnalisation | 10 |
| Légèreté des dépendances | 5 |

Interprétation :

- 85-100 : candidat prioritaire ;
- 70-84 : candidat possible avec adaptations ;
- 55-69 : candidat risqué ;
- moins de 55 : rejet.

## 8. Tests À Faire Avant Achat

Avant d'acheter ou d'adopter un template :

- vérifier la démo desktop ;
- vérifier la démo mobile ;
- inspecter la sidebar ;
- inspecter les tables ;
- inspecter les charts ;
- vérifier les composants de formulaire ;
- vérifier les pages settings ;
- vérifier l'architecture des composants ;
- vérifier la quantité de dépendances ;
- vérifier la licence ;
- vérifier si les exemples sont codés proprement ou seulement visuels.

## 9. Template Idéal Pour Mbàmbulaan

Le template idéal ressemble à une plateforme d'opérations ou de data intelligence, pas à une vitrine.

Il doit pouvoir supporter :

- un ministère qui arbitre ;
- une ONG qui suit un programme ;
- une collectivité qui pilote un territoire ;
- un mareyeur qui organise un flux ;
- une entreprise qui qualifie un risque supply ;
- une organisation qui structure ses membres ;
- un investisseur qui lit une thèse infrastructure.

## 10. Décision Recommandée

Ne pas acheter un template avant d'avoir :

1. validé `docs/07_PRODUCT_UX_BLUEPRINT_FINAL.md` ;
2. scoré au moins trois templates ;
3. vérifié que `/espace-prive` peut être reconstruit naturellement ;
4. confirmé que les pages publiques ne dominent pas l'expérience ;
5. confirmé que le template permet de différencier les espaces par rôle.
