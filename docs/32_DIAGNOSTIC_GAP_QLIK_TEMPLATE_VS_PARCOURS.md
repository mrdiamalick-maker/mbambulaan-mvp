# Diagnostic - Ecart Qlik : template ou parcours ?

## Decision CPO

Le probleme principal n'est pas seulement le template. C'est d'abord le parcours, la direction artistique, la densite d'information et le langage produit.

Un meilleur template peut aider, mais il ne reglera pas seul la perception. Si le parcours reste compose de blocs, cartes et panneaux empiles, le rendu restera loin d'une solution type Qlik.

## Ce qui manque encore

1. Un design system plus strict : grille, espacements, typographie, hierarchie, composants analytics.
2. Une landing plus simple et plus narrative.
3. Un parcours public plus court : comprendre, voir la valeur, demander un essai, se connecter.
4. Des pages demo moins nombreuses ou moins exposees.
5. Des espaces prives plus proches d'une application produit que d'une page admin.
6. Une logique d'analyse progressive : question -> insight -> action.
7. Des visualisations plus fortes : flow, chart, map, score, tendance.
8. Moins de texte, plus de lecture visuelle.

## Template

Le template actuel peut encore servir pour structurer le MVP. Mais il doit etre contraint par une direction produit plus forte.

A court terme : ne pas changer de template maintenant.

A moyen terme : creer un design system Mbambulaan inspire des plateformes analytics B2B, puis remplacer progressivement les composants trop admin.

## Recommandation

Ne pas repartir a zero.

Faire une passe design system et parcours :

- landing Qlik-like ;
- header plus premium ;
- hero plus simple ;
- bloc flow explicable ;
- cas d'usage sous forme de matrice ou diagramme ;
- espace Ministere avec lecture en 3 niveaux : resume, exploration, action ;
- KPIs plus analytiques ;
- cartes et tableaux moins lourds ;
- composants IA plus sobres et rassurants.

## Regle produit

Chaque vue doit repondre a une question.

Exemples :

- Que dois-je savoir maintenant ?
- Quelle zone prioriser ?
- Quel budget pose probleme ?
- Quel programme se chevauche ?
- Quelle ressource est critique ?
- Quelle note puis-je preparer ?

Si une vue ne repond pas a une question claire, elle doit etre simplifiee.

## Prochaine mission Codex

Ne pas demander un changement de template.

Demander une passe : design system + parcours + simplification visuelle + Qlik-like interaction model.

Objectif : transformer l'impression admin en impression solution analytics modulaire.
