# Backlog - Logo, slider et template final

## Decision

Le logo et un slider peuvent apporter une touche premium, mais ils ne doivent pas etre prioritaires avant la stabilisation du parcours produit et du template final.

## Logo

Le logo doit etre traite comme un element de confiance et d'identite, pas comme une simple decoration.

A court terme :

- garder un placeholder propre ;
- eviter de figer une identite trop vite ;
- ne pas multiplier les variantes ;
- utiliser une signature sobre autour de Mbambulaan.

A moyen terme :

- definir une vraie identite visuelle ;
- logo principal ;
- symbole court ;
- favicon ;
- usage clair sur landing, espace prive, documents commerciaux.

## Slider

Un slider peut etre interessant sur la landing, mais seulement s'il raconte une progression produit.

Slider recommande :

1. Cartographie des quais ;
2. KPIs dynamiques par quai ;
3. Referents terrain ;
4. Programmes, budgets et incidents ;
5. IA assistee et note d'arbitrage.

Regle : pas de slider decoratif.

Le slider doit expliquer : vision nationale -> quai -> insight -> action -> trace.

## Timing

Ne pas prioriser maintenant.

Priorite actuelle :

- espace Ministere ;
- dynamique par quai ;
- visualisations analytics ;
- actions qui modifient l'interface ;
- referents terrain ;
- template final.

Le logo et le slider seront traites apres validation du rendu produit et du template final.

## Note Codex

Ne pas lancer de refonte logo maintenant.

Ne pas ajouter de slider tant que la landing et l'espace Ministere ne sont pas stabilises.

Quand le template final sera valide, proposer une passe branding/landing : logo, slider, hero visuel, systeme d'illustrations et assets.
