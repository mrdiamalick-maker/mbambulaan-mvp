# Go-To-Market Playbook — Mbàmbulaan

## 1. Objectif

Ce playbook définit comment Mbàmbulaan passe d’une opportunité institutionnelle ou territoriale à un pilote, puis d’un pilote à un revenu récurrent.

Le principe central est simple :

> Ne pas vendre une application. Vendre une capacité mesurable à mieux coordonner, mieux intervenir et mieux décider.

---

# 2. Cibles prioritaires

## Cible 1 — Institutions publiques

- ministère ;
- directions techniques ;
- services déconcentrés ;
- agences ;
- collectivités.

Besoin principal : visibilité, coordination, suivi et pilotage.

## Cible 2 — Programmes et bailleurs

Besoin principal : traçabilité, suivi-évaluation et preuve d’impact.

## Cible 3 — Organisations professionnelles et structures locales

Besoin principal : continuité des opérations, suivi des équipements et coordination locale.

## Cible 4 — Opérateurs privés

Besoin principal : qualification des demandes, planification et traçabilité des interventions.

---

# 3. Offre d’entrée

L’offre d’entrée doit être un pilote limité et mesurable.

Caractéristiques :

- un site ;
- un problème prioritaire ;
- une durée courte ;
- un nombre réduit d’acteurs ;
- cinq indicateurs maximum ;
- une restitution ;
- une proposition de poursuite.

Le pilote doit être pensé comme un produit commercial, pas comme une expérimentation gratuite sans suite.

---

# 4. Message par cible

## Institution

> Mbàmbulaan vous aide à voir ce qui bloque sur le terrain, à organiser les réponses et à mesurer ce qui a réellement été réglé.

## Programme ou bailleur

> Mbàmbulaan transforme les interventions financées en résultats visibles, suivis et exploitables.

## Organisation locale

> Mbàmbulaan vous aide à faire remonter les difficultés, clarifier les responsabilités et suivre les réponses sans multiplier les démarches.

## Opérateur privé

> Mbàmbulaan vous apporte des demandes mieux qualifiées, un suivi plus clair et une meilleure traçabilité de vos interventions.

---

# 5. Tunnel commercial

## Étape 1 — Identification

Objectif : repérer des organisations confrontées à un problème réel de coordination.

Critères :

- douleur visible ;
- budget ou programme existant ;
- sponsor potentiel ;
- terrain accessible ;
- possibilité de mesurer la valeur.

## Étape 2 — Qualification

Questions :

- quel problème voulez-vous mieux piloter ?
- combien d’acteurs sont concernés ?
- comment l’information circule-t-elle aujourd’hui ?
- quel est le coût du dysfonctionnement ?
- qui décide ?
- qui paierait pour une amélioration ?
- quel budget est mobilisable ?

## Étape 3 — Démonstration

Objectif : faire comprendre Mbàmbulaan en moins de dix minutes.

Séquence :

1. problème ;
2. parcours terrain ;
3. parcours coordinateur ;
4. résultat visible ;
5. valeur institutionnelle ;
6. proposition de pilote.

## Étape 4 — Proposition

Contenu :

- problème ;
- périmètre ;
- acteurs ;
- calendrier ;
- gouvernance ;
- indicateurs ;
- prix ;
- livrables ;
- décision attendue.

## Étape 5 — Négociation

Éléments non négociables :

- périmètre limité ;
- accès au terrain ;
- point focal ;
- données minimales ;
- calendrier ;
- clause de restitution ;
- plan de conversion.

## Étape 6 — Signature

Formes possibles :

- convention ;
- bon de commande ;
- contrat de prestation ;
- marché ;
- financement de pilote ;
- lettre d’engagement.

## Étape 7 — Exécution

Pilotée selon le Mbàmbulaan Operating System.

## Étape 8 — Conversion

Objectif : transformer le pilote en :

- abonnement ;
- licence programme ;
- extension multi-sites ;
- contrat annuel ;
- deuxième pilote financé.

---

# 6. Score de qualification d’une opportunité

Chaque opportunité est notée de 0 à 3 sur les critères suivants :

1. importance du problème ;
2. accès au terrain ;
3. sponsor identifié ;
4. payeur identifié ;
5. budget probable ;
6. valeur mesurable ;
7. réutilisabilité produit ;
8. potentiel de référence.

Interprétation :

- 0 à 8 : faible priorité ;
- 9 à 15 : à approfondir ;
- 16 à 20 : opportunité sérieuse ;
- 21 à 24 : priorité forte.

---

# 7. Discipline de pipeline

Chaque opportunité doit avoir :

- un propriétaire ;
- une prochaine action ;
- une date ;
- un niveau de maturité ;
- un montant potentiel ;
- un risque principal ;
- une probabilité ;
- un sponsor ;
- un payeur.

Aucune opportunité ne doit rester ouverte sans prochaine action datée.

# 9. Objections fréquentes

## « Nous avons déjà WhatsApp »

Réponse : WhatsApp permet de communiquer, mais pas de structurer les responsabilités, suivre les délais, consolider les résultats et piloter plusieurs situations.

## « Nous avons déjà un système d’information »

Réponse : Mbàmbulaan peut compléter les outils existants en organisant la coordination opérationnelle entre acteurs.

## « Les acteurs terrain ne l’utiliseront pas »

Réponse : le produit est construit autour d’actions simples, d’un vocabulaire métier et d’un accompagnement local.

## « Nous n’avons pas de budget »

Réponse : commencer par un périmètre limité, rattacher le pilote à un programme existant ou mobiliser un partenaire financier.

## « Il faut une solution nationale »

Réponse : une solution nationale crédible doit d’abord démontrer sa valeur sur un périmètre limité.

## « Nous voulons tout intégrer dès le départ »

Réponse : une intégration trop large augmente le risque et ralentit la preuve de valeur. Mbàmbulaan recommande une progression par moteurs métier.

# 11. Stratégie d’acquisition

## Canal 1 — Introductions qualifiées

Canal prioritaire pour les décideurs publics et bailleurs.

## Canal 2 — Démonstrations ciblées

Petites sessions centrées sur un problème réel.

## Canal 3 — Événements sectoriels

Objectif : obtenir des rencontres, pas seulement de la visibilité.

## Canal 4 — Contenu d’autorité

Produire :

- notes courtes ;
- analyses de coordination ;
- retours de pilote ;
- indicateurs ;
- cas d’usage.

## Canal 5 — Références

Chaque pilote doit générer :

- une recommandation ;
- une référence ;
- un témoignage ;
- une mise en relation.

# 14. Plan d’action 90 jours

## Jours 1 à 30

- finaliser les supports ;
- identifier dix cibles ;
- obtenir cinq introductions ;
- qualifier trois opportunités ;
- préparer la démonstration.

## Jours 31 à 60

- conduire les démonstrations ;
- identifier un point focal ;
- adapter deux propositions ;
- qualifier un site pilote ;
- sécuriser un partenaire de légitimité.

## Jours 61 à 90

- négocier un pilote ;
- obtenir un engagement écrit ;
- cadrer le budget ;
- préparer les entretiens terrain ;
- sécuriser le plan de conversion.

---

# 15. Règles de refus

Mbàmbulaan doit refuser ou différer une opportunité si :

- aucun problème concret n’est identifié ;
- aucun sponsor n’existe ;
- aucun accès terrain n’est possible ;
- le client exige un produit totalement spécifique ;
- le financement impose une dérive de mission ;
- la gouvernance des données est incompatible ;
- le pilote n’a aucun chemin vers une décision ou un revenu.

---

# 16. Décisions CEO restantes

- seuil minimal de prix d’un pilote ;
- secteurs et partenaires exclus ;
- niveau de remise acceptable ;
- priorité entre institutions, bailleurs et opérateurs privés ;
- politique de gratuité exceptionnelle ;
- niveau de dépendance acceptable à un client majeur ;
- stratégie de marque publique.

---

# Conclusion

Le go-to-market de Mbàmbulaan doit rester sélectif, institutionnel et orienté résultats.

La règle centrale est :

> une bonne opportunité combine un problème réel, un sponsor, un accès terrain, une valeur mesurable et un payeur potentiel.
