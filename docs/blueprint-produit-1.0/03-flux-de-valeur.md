# 03 — Flux de valeur

## 1. Pourquoi raisonner en flux de valeur

Mbàmbulaan ne doit pas être conçu comme une juxtaposition de fonctionnalités. Il doit être conçu comme un système qui transforme des contributions dispersées en décisions, coordinations et résultats utiles.

Un flux de valeur décrit comment une information, une demande, une opportunité, une initiative ou un investissement circule entre plusieurs acteurs jusqu'à produire un résultat mesurable.

La question centrale n'est donc pas :

> Quelle page faut-il construire ?

Mais :

> Quelle valeur doit circuler, entre quels acteurs, avec quelles règles et jusqu'à quel résultat ?

## 2. Flux de valeur fondamentaux

### Flux 1 — Signaler une situation

Un acteur de terrain observe un problème, un besoin, une tension ou une opportunité.

Le flux doit permettre de :

- capter le signal ;
- le qualifier ;
- le localiser ;
- le relier à des acteurs et à un territoire ;
- le valider ;
- le transmettre aux bons décideurs ;
- suivre la réponse apportée.

Exemples :

- rupture de glace ;
- baisse inhabituelle des débarquements ;
- problème sanitaire ;
- besoin de formation ;
- opportunité de financement ;
- infrastructure dégradée.

### Flux 2 — Transformer un besoin en initiative

Un besoin reconnu doit pouvoir devenir une initiative structurée.

Le flux doit relier :

- le problème ;
- les bénéficiaires ;
- les objectifs ;
- les responsables ;
- les partenaires ;
- les ressources ;
- les risques ;
- les indicateurs ;
- les résultats attendus.

Ce flux évite que les besoins restent de simples constats et que les projets apparaissent sans justification claire.

### Flux 3 — Transformer une initiative en financement

Une initiative crédible doit pouvoir être présentée à un financeur public, privé ou de développement.

Le flux doit permettre de :

- consolider le dossier ;
- démontrer le besoin ;
- estimer les coûts ;
- documenter les bénéficiaires ;
- expliciter les résultats attendus ;
- identifier les cofinancements ;
- suivre les engagements ;
- tracer les décaissements et résultats associés.

### Flux 4 — Transformer une information en décision

La donnée brute ne crée pas de valeur tant qu'elle n'est pas contextualisée.

Le flux doit relier :

- l'observation ;
- les tendances ;
- les comparaisons ;
- les causes probables ;
- les acteurs concernés ;
- les options d'action ;
- la décision ;
- le suivi de ses effets.

### Flux 5 — Transformer une action en apprentissage

Chaque action achevée doit enrichir la mémoire de la filière.

Le flux doit documenter :

- ce qui a été fait ;
- dans quel contexte ;
- avec quels moyens ;
- ce qui a fonctionné ;
- ce qui a échoué ;
- les conditions de réussite ;
- les enseignements ;
- la possibilité de reproduction.

### Flux 6 — Transformer une contribution en retour de valeur

Aucun acteur ne doit seulement fournir des données.

Toute contribution doit produire un retour proportionné :

- information utile ;
- visibilité ;
- accès à une opportunité ;
- reconnaissance ;
- recommandation ;
- service ;
- amélioration de coordination ;
- décision plus rapide.

## 3. Objets de coordination

Les flux de valeur s'organisent autour d'objets métier partagés.

Les principaux objets sont :

- acteur ;
- organisation ;
- territoire ;
- site ou infrastructure ;
- situation ;
- besoin ;
- initiative ;
- portefeuille de transformation ;
- financement ;
- engagement ;
- action ;
- indicateur ;
- résultat ;
- apprentissage ;
- ressource documentaire ;
- contribution ;
- décision.

Ces objets ne doivent pas être isolés. La valeur provient de leurs relations.

Exemple :

```text
Situation territoriale
        ↓
Besoin qualifié
        ↓
Initiative
        ↓
Financement
        ↓
Actions
        ↓
Résultats
        ↓
Apprentissages
        ↓
Recommandations futures
```

## 4. Cycle de vie commun

Tous les objets structurants doivent suivre un cycle de vie explicite.

États génériques :

1. proposé ;
2. à qualifier ;
3. validé ;
4. priorisé ;
5. en préparation ;
6. en cours ;
7. bloqué ou à risque ;
8. terminé ;
9. évalué ;
10. capitalisé ou archivé.

Chaque transition doit préciser :

- qui peut agir ;
- sur quelle base ;
- quelles preuves sont requises ;
- qui doit être informé ;
- quelle trace est conservée.

## 5. Valeur créée par acteur

| Acteur | Contribution principale | Valeur reçue |
|---|---|---|
| Pêcheur / acteur terrain | Signal, donnée opérationnelle, retour d'expérience | Information utile, services, opportunités, réponse coordonnée |
| CLPA / collectivité | Qualification locale, priorisation, mobilisation | Vision territoriale, dossiers crédibles, partenaires |
| Ministère | Arbitrage, référentiels, coordination institutionnelle | Pilotage consolidé, redevabilité, meilleure allocation |
| Bailleur / ONG | Financement, expertise, documentation | Pipeline qualifié, suivi d'impact, capitalisation |
| Banque / investisseur | Financement, critères de risque | Opportunités mieux documentées, réduction d'asymétrie |
| Chercheur / expert | Analyse, connaissance, méthode | Accès à des situations, diffusion et impact des travaux |
| Mbàmbulaan | Orchestration, fiabilisation, traçabilité | Revenus, données gouvernées, avantage de réseau |

## 6. Flux prioritaires pour la version 1.0

La version 1.0 ne doit pas couvrir tous les flux à profondeur égale.

Les flux prioritaires sont :

1. signaler et qualifier une situation territoriale ;
2. structurer une initiative à partir d'un besoin ;
3. coordonner les acteurs et engagements autour de cette initiative ;
4. suivre les actions, risques et résultats ;
5. documenter les apprentissages ;
6. produire une vue territoriale et une vue portefeuille.

Le financement transactionnel, la marketplace et les services financiers intégrés ne sont pas prioritaires dans la première version. Ils pourront être connectés une fois que la qualité des dossiers, la confiance et la coordination auront été démontrées.

## 7. Mesure de performance d'un flux

Un flux de valeur est performant lorsqu'il réduit :

- le temps entre signalement et prise en charge ;
- les doublons ;
- les informations manquantes ;
- les relances manuelles ;
- les décisions sans preuve ;
- les initiatives sans responsable ;
- les financements sans résultat traçable.

Et lorsqu'il augmente :

- la complétude ;
- la fiabilité ;
- la participation ;
- la vitesse de coordination ;
- le taux de transformation des besoins en actions ;
- la part des initiatives évaluées ;
- la réutilisation des apprentissages.

## 8. Principe de conception

> Mbàmbulaan ne doit jamais ajouter une étape administrative sans supprimer une friction, améliorer une décision ou renforcer la confiance.
